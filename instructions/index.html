CS2 Farm — инструкция для клиента

Документ для самостоятельной установки и запуска фермы на хосте + 4 VM.

Реализовано: 2x2 Пока не реализовано: 5x5 Пока не реализовано: счет random / 8:8 в farm-flow

Этап 0. Предусловия

  1. Подготовить 1 хост + 4 VM (Windows) в одной сети.
  2. На каждой VM установить Steam и CS2.
  3. Подготовить аккаунты (по одному аккаунту на VM).
  4. На хосте подготовить:
    • logpass.txt (формат: login:password);
    • папку maFiles с .maFile (если используется 2FA);
    • config.json с корректными путями и IP VM.

Этап 1. Настройка VM1–VM4

  1. Скопировать папку _client на каждую VM в C:\CS2_FARM.
  2. Рекомендуемый вариант установки:
    • запустить install_vm.bat от администратора;
    • скрипт ставит Python, Node.js, Python-пакеты и выполняет npm install.
  3. Если вручную:
    cd C:\CS2_FARM npm install python -m pip install requests
  4. Запустить агент:
    • run_agent.bat (рекомендуется), или
    • python vm_agent.py.
  5. Проверка: агент слушает 0.0.0.0:9999.

Этап 2. Настройка хоста

  1. Установить зависимости на хосте:
    python -m pip install -r requirements.txt
  2. Заполнить данные аккаунтов:
    • в logpass.txt добавить аккаунты в формате login:password;
    • в maFiles загрузить .maFile для 2FA.
  3. Проверить config.json:
    • блок vms содержит 4 VM с корректными ip и port=9999;
    • steam_path и cs2_path валидны для VM.
  4. Открыть панель:
    python cs_2_farm_panel_ui_modern.py
  5. Назначить аккаунты на VM в блоке Players.

Важно про кнопку Check Connection

В текущей версии кнопка check connection является заглушкой и не выполняет реальную сетевую проверку.

Проверять доступность VM рекомендуется так:

  • смотреть статусы VM в панели (опрос /status), или
  • проверять вручную URL: http://IP_VM:9999/status.

Сеть и Firewall (порт 9999)

  • На каждой VM открыть TCP 9999 во входящих правилах (Inbound, Allow).
  • Рекомендуется также открыть TCP 9999 в исходящих правилах на VM (Outbound, Allow).
  • На хосте отдельное открытие 9999 обычно не требуется, но исходящие подключения к VM не должны блокироваться.
  • Проверка с хоста: http://IP_VM:9999/status должен отдавать JSON-ответ.

Этап 3. Запуск фермы

  1. Нажать Запустить 2v2 — логин и запуск CS2 на всех 4 VM.
  2. После загрузки CS2 нажать Старт Лобби — этот шаг включает приглашение напарника в группу и запуск поиска общей карты; дальнейшие действия пока не реализованы.
  3. Контролировать процесс по логу панели и статусам VM.
  4. Для остановки использовать Стоп (Logout).

Что поддерживается сейчас

  • Поддерживается: 2x2 на 4 VM.
  • Пока не поддерживается: 5x5 как режим фермы.
  • Пока не поддерживается: логика счета random / 8:8 в основном farm-сценарии.
  • Сценарии поведения: в основной ферме пока не интегрированы (будут добавляться позже).
