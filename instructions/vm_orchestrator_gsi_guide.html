VM Orchestrator по GSI

Полный цикл: хост управляет матч-сессией через de_poseidon_host_orchestration.json (цель по счету, план, профиль), а VM принимает решение по сценарию в раунде локально на основе GSI (team + round) и runtime-конфига de_poseidon_vm_runtime.json.

Полный цикл (end-to-end)

  1. Хост запускает 2v2 сессию и получает подтверждение, что все VM в одном матче.
  2. Хост выбирает профиль сессии: карта, plan_id, целевой итог (например, 8:8).
  3. Хост отправляет на каждую VM контекст сессии: slot, plan_id, map.
  4. VM-оркестратор читает локальный GSI и определяет текущие team и round.
  5. VM выбирает сценарий через plan_id + team + slot + round из JSON-плана.
  6. Хост для каждого нового раунда решает целевого победителя (T или CT) для выхода к целевому счету.
  7. Хост отправляет в VM planned_winner (и при необходимости skill_policy_id).
  8. VM применяет skill-политику: стороне-победителю назначает высокий профиль, проигрывающей — низкий.
  9. VM запускает/перезапускает nav_demo.py (перемещение) и main.py (стрельба), логирует решение и отдает статус хосту.

1. Что отправляет хост на старте

Хост отправляет VM контекст сессии, а не конкретный сценарий:

Ниже показан пример в формате JSONC (с комментариями для чтения).

{
  "map": "de_poseidon",                 // карта матча
  "plan_id": "plan_balanced_v1",        // id плана ротации сценариев
  "slot": "slot1",                      // роль VM в паре (slot1|slot2)
  "target_score": { "T": 8, "CT": 8 },  // целевой итог матча
  "skill_policy_id": "winner_loser_policy" // политика мастерства winner/loser
}

slot1/slot2 — постоянная роль VM в паре (например VM1/VM3 = slot1, VM2/VM4 = slot2).

2. Откуда VM берет сценарий

  1. VM читает GSI: team, round.
  2. Берет список сценариев из plan_id + team + slot.
  3. Считает индекс: idx = (round - 1) % len(slot_list).
  4. Запускает nav_demo.py с выбранным --scenario для перемещения.
  5. Параллельно применяет профиль мастерства к main.py (контур стрельбы/аима).

2. Команды с хоста на VM

Рекомендуется отправлять не прямую команду сценария каждый раунд, а команды управления сессией. VM сама выберет нужный сценарий по GSI.

1) Старт bot-сессии

Пример JSONC (с комментариями для оператора):

POST /bot/session/start
{
  "map": "de_poseidon",                 // карта для nav_demo
  "plan_id": "plan_balanced_v1",        // какой план из vm_orchestrated.json использовать
  "slot": "slot1",                      // slot этой VM
  "bot_root": "C:\\CS2_FARM\\bot",      // корень, где лежат nav_demo.py и main.py
  "target_score": { "T": 8, "CT": 8 },  // цель по итоговому счету
  "skill_policy_id": "winner_loser_policy" // правило выдачи skill_profile
}

Назначение: запустить локальный оркестратор на VM и привязать его к профилю плана.

2) Смена профиля в живой сессии (опционально)

POST /bot/session/update
{
  "plan_id": "plan_fast_v1",            // переключение ротации сценариев
  "skill_policy_id": "winner_loser_policy" // переключение политики навыка
}

3) Команда на целевой исход текущего раунда

POST /bot/round/target
{
  "round": 6,               // для какого раунда отправлено решение
  "planned_winner": "T"     // кто должен выиграть: "T" или "CT"
}

Назначение: хост задает, кто должен выиграть раунд для движения к целевому счету.

4) Остановка bot-сессии

POST /bot/session/stop
{}

Назначение: остановить оркестратор и завершить запущенные процессы nav_demo.py и main.py.

5) Запрос текущего состояния

GET /bot/status

{
  "running": true,                          // оркестратор активен
  "map": "de_poseidon",                     // текущая карта
  "plan_id": "plan_balanced_v1",            // активный план
  "slot": "slot1",                          // роль VM в текущей сессии
  "current_team": "T",                      // сторона из GSI
  "current_round": 4,                       // раунд из GSI
  "current_scenario": "eco_delay",          // выбранный сценарий на этот раунд
  "planned_winner": "T",                    // цель от хоста
  "applied_skill_profile": "win_headshot_only", // примененный профиль стрельбы
  "last_switch_ts": "2026-04-14T16:32:12"   // время последнего переключения
}

Для реальных HTTP-пакетов отправляй обычный JSON без комментариев.

3. Как хост решает победителя раунда (для 8:8)

Хост ведет глобальный счет и на каждом раунде пересчитывает, кому нужно отдать победу:

need_t  = target_t  - current_t
need_ct = target_ct - current_ct

if need_t > need_ct:
    planned_winner = "T"
elif need_ct > need_t:
    planned_winner = "CT"
else:
    planned_winner = alternate_or_seeded_random()

После вычисления хост отправляет POST /bot/round/target на каждую VM.

4. Минимальный псевдокод VM-решения

# VM side round orchestrator
config = load_json("scenarios/de_poseidon_vm_runtime.json")
state = {
    "last_round": None,
    "last_team": None,
    "last_scenario": None,
    "bot_proc": None,
}

vm_slot = "slot1"
plan_id = "plan_balanced_v1"
map_name = "de_poseidon"

while session_active:
    gsi = read_latest_gsi()  # {"team": "T"|"CT", "round": int}
    team = gsi.get("team")
    rnd = gsi.get("round")

    if team not in ("T", "CT") or not isinstance(rnd, int) or rnd <= 0:
        sleep(0.25)
        continue

    if rnd == state["last_round"] and team == state["last_team"]:
        sleep(0.25)
        continue

    slot_list = config["vm_selection"]["plans"][plan_id][team][vm_slot]
    idx = (rnd - 1) % len(slot_list)
    scenario = slot_list[idx]
    winner = state.get("planned_winner")  # от /bot/round/target

    if winner == "T":
        skill_profile = config["skill_model"]["winner_loser_policy"]["when_T_should_win"][team][vm_slot]
    elif winner == "CT":
        skill_profile = config["skill_model"]["winner_loser_policy"]["when_CT_should_win"][team][vm_slot]
    else:
        skill_profile = config["skill_model"]["winner_loser_policy"]["fallback_balanced"][team][vm_slot]

    if scenario != state["last_scenario"]:
        stop_process(state["nav_proc"])
        state["nav_proc"] = start_process([
            "python",
            r"C:\\CS2_FARM\\bot\\nav_demo.py",
            "--map", map_name,
            "--team", team,
            "--scenario", scenario,
        ])
        stop_process(state["aim_proc"])
        state["aim_proc"] = start_process([
            "python",
            r"C:\\CS2_FARM\\bot\\main.py",
            "--team", team,
            # profile -> runtime config bridge (через env/config/IPC)
        ])

    state["last_round"] = rnd
    state["last_team"] = team
    state["last_scenario"] = scenario
    sleep(0.25)

5. Пример команд, которые VM запускает

# 1) Перемещение (сценарий)
python C:\CS2_FARM\bot\nav_demo.py --map de_poseidon --team T --scenario rush_a

# 2) Стрельба / аим (отдельный процесс)
python C:\CS2_FARM\bot\main.py --team T

Для CT аналогично, только --team CT. Профиль мастерства применяется к контуру стрельбы (main.py), а не к nav_demo.py.

6. Практические защиты

  • Debounce на смену раунда: 0.5-1.0 сек.
  • Graceful stop старого процесса (2 сек), затем принудительный stop при необходимости.
  • Если GSI временно пропал — не менять сценарий, держать последний рабочий.
  • Если от хоста не пришел planned_winner — использовать fallback_balanced.
  • Логировать: round, team, slot, scenario, skill_profile, nav_proc, aim_proc.
